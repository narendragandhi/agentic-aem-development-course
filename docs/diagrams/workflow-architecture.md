## Workflow Launcher Architecture

```
┌───────────────────────────────────────────────────────────────────────────────┐
│                      WORKFLOW LAUNCHER ARCHITECTURE                            │
├───────────────────────────────────────────────────────────────────────────────┤
│                                                                               │
│   ┌─────────────────────────────────────────────────────────────────────────┐│
│   │                        JCR OBSERVATION                                  ││
│   │                                                                         ││
│   │     JCR Repository monitors for:                                        ││
│   │     • NODE_ADDED (CREATED)                                              ││
│   │     • PROPERTY_CHANGED (MODIFIED)                                       ││
│   │     • NODE_REMOVED (DELETED)                                            ││
│   │                                                                         ││
│   └───────────────────────────────────┬─────────────────────────────────────┘│
│                                       │                                      │
│                                       ▼                                      │
│   ┌─────────────────────────────────────────────────────────────────────────┐│
│   │                     LAUNCHER EVALUATION                                 ││
│   │                                                                         ││
│   │   For each launcher configuration:                                      ││
│   │                                                                         ││
│   │   ┌───────────────────────────────────────────────────────────────┐    ││
│   │   │  1. CHECK EVENT TYPE                                          │    ││
│   │   │     Does event match launcher's eventType?                    │    ││
│   │   │     (CREATED, MODIFIED, DELETED)                              │    ││
│   │   └───────────────────────────────────────────────────────────────┘    ││
│   │                              │                                         ││
│   │                              ▼                                         ││
│   │   ┌───────────────────────────────────────────────────────────────┐    ││
│   │   │  2. CHECK PATH PATTERN                                        │    ││
│   │   │     Does affected path match launcher's glob pattern?         │    ││
│   │   │     e.g., /content/dam/demo-workflow/.*                       │    ││
│   │   └───────────────────────────────────────────────────────────────┘    ││
│   │                              │                                         ││
│   │                              ▼                                         ││
│   │   ┌───────────────────────────────────────────────────────────────┐    ││
│   │   │  3. CHECK NODE TYPE                                           │    ││
│   │   │     Does node type match launcher's nodetype?                 │    ││
│   │   │     e.g., dam:Asset, cq:PageContent                           │    ││
│   │   └───────────────────────────────────────────────────────────────┘    ││
│   │                              │                                         ││
│   │                              ▼                                         ││
│   │   ┌───────────────────────────────────────────────────────────────┐    ││
│   │   │  4. CHECK CONDITIONS                                          │    ││
│   │   │     Do additional conditions evaluate to true?                │    ││
│   │   │     e.g., [cq:workflowStatus=pending-approval]                │    ││
│   │   └───────────────────────────────────────────────────────────────┘    ││
│   │                              │                                         ││
│   │                              ▼                                         ││
│   │   ┌───────────────────────────────────────────────────────────────┐    ││
│   │   │  5. CHECK RUN MODE                                            │    ││
│   │   │     Is current run mode included in launcher's runModes?      │    ││
│   │   │     e.g., author, publish                                     │    ││
│   │   └───────────────────────────────────────────────────────────────┘    ││
│   │                                                                         ││
│   └───────────────────────────────────┬─────────────────────────────────────┘│
│                                       │                                      │
│                                       ▼                                      │

│   │                     WORKFLOW INSTANTIATION                              ││
│   │                                                                         ││
│   │   ┌───────────────────────────────────────────────────────────────┐    ││
│   │   │                                                               │    ││
│   │   │   • Load workflow model from launcher configuration           │    ││
```
